# Page Recipes

# Landing Page Recipe

Accomplish: explain the offer, build trust, and move the right user to the next action.

Required hierarchy: concrete headline, specific supporting copy, primary CTA, proof, product/offer detail, objections, final CTA.

Avoid vague SaaS copy, generic gradient heroes, three identical feature cards, no proof, and visual style unrelated to the product.

# Dashboard Recipe

Accomplish: help users understand status, spot change, and act.

Required hierarchy: time range/context, dominant insight or exception, supporting KPIs, chart/table context, prioritized work queue, drill-down actions, empty/error/loading states.

Avoid equal KPI cards only, unlabeled charts, missing time ranges, no explanation of change, and dashboards that decorate data instead of supporting decisions.

# Settings Page Recipe

Accomplish: help users configure safely.

Required hierarchy: category navigation, grouped controls, clear labels/help, save/cancel or autosave status, separated danger zone.

Required states: saved, unsaved, saving, error, disabled, destructive confirmation.

Avoid unclear save behavior, destructive actions near normal actions, placeholder-only labels, and dense ungrouped forms.

# Pricing Page Recipe

Accomplish: make plan differences clear and reduce purchase anxiety.

Required hierarchy: value headline, billing toggle if needed, differentiated plan cards, recommended plan if appropriate, comparison, trust/reassurance, FAQ.

Required states: selected billing interval, loading price if dynamic, unavailable plan, enterprise/contact path.

Avoid identical plans, unclear limits, hidden fees, vague feature names, too many primary CTAs.

# Checkout Flow Recipe

Accomplish: complete purchase confidently, reduce mistakes, preserve trust.

Required hierarchy: order summary, contact details, payment/shipping details, trust/security signals, final review, clear submit.

Required states: validation, processing, payment error, success, recoverable failure, disabled submit with reason.

Avoid surprising fees, losing form data, vague payment errors, aggressive upsells, and clutter near payment.

# Auth Flow Recipe

Accomplish: sign in/up/reset quickly while preserving security and recovery.

Required hierarchy: clear auth purpose, persistent labels, primary action, recovery links, alternate auth methods, useful trust note.

Required states: invalid credentials, loading, locked/rate-limited, reset sent, password requirements, account exists/does not exist.

Avoid placeholder-only labels, vague errors, disabling paste, breaking password managers, and hidden recovery paths.

# Onboarding Flow Recipe

Accomplish: get to first value quickly and collect only necessary information.

Required hierarchy: value reminder, current step, minimum required action, skip/back where appropriate, progress, completion.

Required states: step loading, validation error, saved progress, skipped step, completion.

Avoid long forms before value, no progress, forced irrelevant personalization, unclear skip behavior.

# Data Table Recipe

Accomplish: let users scan, compare, sort, filter, and act on records.

Required hierarchy: table title/context, search/filter/sort controls, active filters, clear columns, row actions, pagination/virtualization, empty/error states.

Responsive concerns: collapse less important fields, use cards only if comparison is not central, allow horizontal scroll for true tabular data, keep actions accessible.

Avoid truncating critical data, hidden sort state, filters with no reset, no empty state, hover-only row actions, unreadable dense mobile tables.

# Admin Panel Recipe

Accomplish: support efficient repeat work and reliability.

Required hierarchy: clear page title, status/context, primary admin action, filters/search, work queue/table, bulk actions, audit/status feedback.

Avoid marketing-style heroes, oversized whitespace, decorative cards everywhere, unclear permissions, destructive actions without confirmation.

# Mobile-First App Screen Recipe

Accomplish: make the main task thumb-friendly and obvious.

Required hierarchy: compact header/context, main content/action, secondary details, reachable primary action, navigation that does not hide critical tasks.

Avoid tiny targets, hover-only interactions, fixed desktop widths, sticky elements covering content, long ungrouped forms, and modals that do not fit mobile height.
